Développement d'une WebApp pour Firefox OS

Amélie Lefèvre, Charlotte Saintpierre


L'application

Cette application est une Webapp développée pour Firefox OS. Cette application est en fait un jeu qui teste les connaissances musicales d'un utilisateur sur sa bibliothèque musicale.


Etude de l'existant

En ce qui concerne ce projet, nous n'avons pas trouvé de projet existant sur le Marketplace de Firefox. Nous avons donc créé entièrement l'application.


Les outils et les langages utilisés

Afin de tester l'application ainsi que les différentes étapes de développement, nous avons utilisé un simulateur. La version de Firefox OS sur le simulateur est la 1.1 et la version du simulateur est la 4.0.1. Les langages utilisés pour ce jeu sont les suivant : le HTML, le CSS et le JavaScript.


Les fonctionnalités

Le principe du jeu est très simple : l'application questionne l'utilisateur à propos des extraits musicaux, choisit aléatoirement, qu'il entend. Une réponse correcte apporte des points, le but étant d'avoir le score le plus élevé possible.

Il y a six niveaux de difficultés regroupés en deux modes le « Zen Mode » et le « Compet Mode ». Le premier mode est un mode sans fin : c'est le « Zen Mode ». Le second joue dix extraits de musique. Dans chacun des deux modes, il y a trois niveaux : « Level 1 », « Level 2 » et « Level 3 ». Les règles du jeu pour les niveaux sont les mêmes dans les deux modes. C'est-à-dire que l'extrait est joué 30 secondes pour le « Level 1 » du « Zen Mode » et du « Compet Mode » et l'utilisateur doit uniquement donner le titre de l'extrait.

On a donc des difficultés croissantes en fonction du niveau. Le « Level 2 » joue un extrait durant 25 secondes, l'utilisateur doit uniquement saisir le chanteur. Le « Level 3 » joue un extrait pendant 20 secondes, l'utilisateur doit donner le titre ainsi que le chanteur.

Une des autres fonctionnalités permet à l'utilisateur de regarder ses cinq meilleurs scores.

L'application vérifie la saisie de l'utilisateur et incrémente le score à chaque réponse juste.


Les problèmes rencontrés

Nous avons eu quelques problèmes en ce qui concerne la lecture de la musique par cette version du simulateur. Après conversion des fichiers audio dans le format souhaité, la lecture de la musique n'a plus posé de problème.


Les limites et améliorations

Une des amélioration possible serait d'empêcher le mode aléatoire de rejouer une musique déjà écoutée par l'utilisateur. Il serait également intéressant de proposer à l'utilisateur de la musique hébergée sur un serveur distant. C'est-à-dire de la musique qu'il n'aurait pas stockée sur son téléphone. On pourrait également proposer d'autres niveaux de difficultés.

En ce qui concerne les limites, la carte sd du téléphone offre peut d'espace de stockage.
